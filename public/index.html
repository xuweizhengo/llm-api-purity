后端真实探针

每个检测项都会发起真实 API 请求，并用响应结构与事件序列生成证据。

Key 不入库

API Key 只用于本次请求，错误信息会脱敏，报告里只展示接口哈希。

状态链路验证

OpenAI 检测 Responses / Chat，Claude 检测 Messages / SSE / tool_use。

成本倍率审计

用多组短请求检查 usage、cached tokens 与实际用量的稳定性。

为什么需要纯度检测

中转站越多，接口“可用”和“纯净”越不能画等号

市场上的 API 中转服务质量参差不齐。一个接口能返回文本，并不代表它完整保留了官方协议、 工具调用、流式生命周期和 Token 用量统计。PureAPI Radar 通过多探针验证，帮助你识别隐藏风险。

模型掺杂 同一个模型名下可能被替换成别的模型，回答质量和工具调用表现会变化。
协议残缺 Responses、Messages、tool_use、SSE 等结构可能只做了部分兼容。
异常来源 共享缓存、状态链断裂、错误信息不规范会影响生产稳定性。
请求看似成功 真实来源不明 后端探针逐项验证后再给结论

OpenAI 检测链路

ModelsResponsesTool CallSSEAudit
  • GET /v1/models 验证 list 结构与模型可见性。
  • POST /v1/responses 或 /v1/chat/completions 验证结构。
  • tool_choice 要求模型返回 function tool_calls。
  • SSE 验证 delta 与 DONE 生命周期。
  • Token 审计使用 prompt、completion 与 cached tokens。

Claude 检测链路

Messagestool_useSSEImageAudit
  • POST /v1/messages 验证 message 结构和 stop_reason。
  • tool_choice 指向 probe_ping，确认真实 tool_use 返回。
  • SSE 验证 message_start、content_block_delta、message_stop。
  • 多模态使用 image block 检测 Messages 兼容性。
  • Token 审计使用 cache_creation 与 cache_read 字段。
